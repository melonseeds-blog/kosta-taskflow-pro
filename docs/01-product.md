# 01. Product — 제품 정의

## 한 줄 목표
> **팀 업무를 시각화하여 "지금 누가 뭐 해?"라는 질문이 사라지게 한다.**

## 페르소나
- **타깃**: 10명 규모 스타트업의 팀 리더
- **연령대**: 30~40대
- **상황**: 매일 슬랙·구두 보고로 팀원 진행 상황을 파악 중. 누락·중복·블로커가 잦다.
- **니즈**: 한 화면에서 "누가 / 무엇을 / 언제까지" 를 즉시 본다.

---

## MVP 범위 (이번 단계에서 만든다)

### 기능
- **CRUD 4종 모두 화면에서 동작**
  - 추가 / 목록 / 수정 / 삭제
- **상태 분류 + 마감 시각 (`due_at`) 지정**
  - 날짜만이 아니라 **시간**까지 (예: `2026-05-12 18:00`)

### 비기능
- **라이트/다크 테마 토글** (`localStorage` 영속화)
- **모바일 반응형** — 최소 폭 **360px** 에서 깨지지 않음

### UI 톤 — Mac OS 스타일
- 둥근 모서리 (rounded)
- 부드러운 그림자 (soft shadow)
- 반투명 카드 (translucent / backdrop blur)
- 시스템 폰트 (`-apple-system`, `Segoe UI`, …)

---

## 확장 범위 (다음 단계 후보)
- JWT 로그인
- 팀(멀티 테넌시)
- Kanban 보드
- 팀 채팅
- CI/CD 파이프라인

> 확장 항목은 MVP 성공 기준을 모두 충족한 뒤 착수한다.

---

## 범위 외 (이번 프로젝트에서 다루지 않음)
- 외부 결제
- 네이티브 앱 (iOS/Android)
- WebRTC (음성·영상)
- 외부 캘린더 연동
- 파일 업로드
- **WebSocket** — MVP는 **폴링**으로 처리. WebSocket은 확장 단계에서도 보류.

---

## 성공 기준 (MVP Done의 정의)
1. **새로고침 후에도 데이터·테마가 유지**된다.
2. **360px 폭에서 레이아웃이 깨지지 않는다.**
3. **API 응답 200ms 이내** (로컬·소규모 데이터 기준).
4. **CRUD 4종이 화면에서 모두 동작**한다.
5. **테마 토글이 즉시 반영**되고 새로고침 후에도 유지된다.

> 5개 기준 중 하나라도 미달이면 MVP 미완으로 본다.

---

## MVP 성공 기준 검증 결과 (2026-05-14)

| # | 기준 | 검증 방법 | 결과 |
|---|---|---|---|
| 1 | 데이터·테마 새로고침 유지 | SQLite 파일 영속화 + `localStorage('theme')` 영속화. 라이브 서버에서 `POST` 후 `GET` 200 확인. | ✅ 통과 |
| 2 | 360px 레이아웃 미파괴 | Tailwind 모바일 우선 + `flex-col sm:flex-row` 분기. 코드/구조 확인. | 🔵 코드 통과 / 사용자 브라우저 시각 확인 |
| 3 | API 응답 200ms 이내 | PowerShell `Stopwatch` 5회 평균 측정 (워밍업 후). | ✅ 통과 (전 엔드포인트 max ≤ 102.5ms) |
| 4 | CRUD 4종 화면 동작 | 5개 엔드포인트 라이브 호출 검증 + 프론트 코드 작성 완료. | ✅ API 통과 / 사용자 브라우저 시각 확인 |
| 5 | 테마 토글 즉시·영속 | FOUC 방지 인라인 스크립트 + `localStorage('theme')`. 코드 점검 완료. | 🔵 코드 통과 / 사용자 브라우저 시각 확인 |

### 응답 시간 측정 상세 (ms, 5회 측정)

| 엔드포인트 | min | avg | max | 기준 |
|---|---|---|---|---|
| `POST /api/tasks` | 82.3 | 86.9 | 88.8 | < 200 ✅ |
| `GET /api/tasks` (목록) | 7.7 | 8.2 | 9.7 | < 200 ✅ |
| `GET /api/tasks/{id}` | 7.6 | 7.9 | 8.4 | < 200 ✅ |
| `PUT /api/tasks/{id}` | 8.7 | 9.9 | 11.1 | < 200 ✅ |
| `DELETE /api/tasks/{id}` | 82.1 | 91.5 | 102.5 | < 200 ✅ |

POST/DELETE 가 ~90ms 인 이유는 SQLite WAL 커밋·fsync 비용이며, MVP 기준(200ms) 내 안정 확보.

### 종합
자동 검증 항목 3종(데이터 영속화 · API 응답 시간 · CRUD API)은 통과. 시각·반응형 항목 2종(360px · 테마 토글 시각)은 코드 정합성 확인 + 사용자 브라우저 검증으로 대체.
